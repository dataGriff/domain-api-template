    📄 Data Contract

    Items Domain Data Contract v1.0.0

    Defines the structure and quality expectations for item data published by the Items domain via domain events. This contract is the authoritative historical integration point for downstream consumers.

    items

    Item records published via the items.item.added and items.item.edited event channels. Each record represents the current state of an item at the time the event was emitted.

    FieldLogical TypePhysical TypeRequiredDescription
    id 🔑stringuuid✓Unique identifier for the item.
    namestring✓Display name of the item.
    descriptionstringOptional longer description of the item.
    statusstring✓Lifecycle status of the item — either active or archived.
    contributorIdstringuuid✓Identifier of the contributor who added the item.
    createdAttimestamp✓Timestamp when the item was added to the catalogue.
    updatedAttimestamp✓Timestamp of the most recent edit to the item.
